# WhisperType

Windows向け AIリアルタイム音声入力アプリ 要件定義（Ver.1.0）

## 1. 概要

### 1.1 アプリ名

WhisperType

### 1.2 コンセプト

WhisperTypeは、「WindowsでiPhoneのような快適な音声入力を実現する」ことを目的としたAI音声入力アプリケーションである。

ユーザーはホットキーを押すだけで音声入力を開始でき、話した内容はリアルタイムに文字へ変換され、現在カーソルがある位置へ自動入力される。

本アプリはバックグラウンドで常駐し、Windows上のあらゆるアプリケーションから利用できることを目標とする。

すべての音声認識はローカルPC上で実行し、音声データを外部へ送信しない。

## 2. 開発環境

| 項目 | 内容 |
|---|---|
| OS | Windows 11 |
| プログラミング言語 | Python 3.11以上 |
| GUI | PySide6 |
| 音声認識エンジン | whisper.cpp |
| Whisperモデル | large-v3-turbo（GGUF形式） |
| 音声録音 | sounddevice |
| 自動入力 | pynput |
| グローバルホットキー | keyboard |
| 数値処理 | NumPy |

## 3. システム構成

```
                マイク
                  │
                  ▼
         sounddevice（音声取得）
                  │
                  ▼

                  │
                  ▼

```

## 4. 機能要件

### 4.1 常駐機能

本アプリはWindows起動後もバックグラウンドで動作する。

機能:

- システムトレイへ常駐
- メイン画面の表示・非表示
- アプリ終了
- 起動時に自動常駐（設定で変更可能）

### 4.2 グローバルホットキー

初期設定: `Ctrl + Shift + Space`

ホットキーはユーザーが変更できる。

動作:

- 1回押す：音声入力開始
- もう一度押す：音声入力停止

### 4.3 リアルタイム音声認識

whisper.cppを利用してストリーミング音声認識を実行する。

使用モデル:

- large-v3-turbo

要件:

- 日本語対応
- 英語対応
- リアルタイム認識
- 途中認識結果を継続的に更新
- ローカル実行
- オフライン利用可能（モデル配置済みの場合）

### 4.4 自動入力

認識結果を現在フォーカスされているアプリケーションへ入力する。

使用ライブラリ: pynput

対応例:

- メモ帳
- Microsoft Word
- Microsoft Excel
- Outlook
- Teams
- Slack
- Webブラウザー
- ChatGPT
- Visual Studio Code

認識結果はリアルタイムに入力される。

### 4.5 入力制御

設定により以下を切り替えられる。

- 自動入力 ON / OFF
- 自動入力開始までの待機時間
- 入力速度

### 4.6 認識状態表示

GUIには以下を表示する。

- 待機中
- 音声入力中
- 認識中
- エラー

## 7. 非機能要件

- Windowsネイティブアプリとして動作すること
- Pythonのみで開発すること
- ローカルPCのみで処理すること
- 音声データを外部へ送信しないこと
- オフライン利用可能であること（モデル配置済みの場合）
- CPU環境でも快適に動作すること
- GPUが利用可能な場合は性能を活用できる構成を検討すること
- メモリ使用量を抑えること
- 長時間常駐しても安定して動作すること
